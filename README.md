## Implementation Steps

### 1. Start the Minikube Cluster

```bash
minikube start --driver=docker
minikube status
```

---

### 2. Deploy Kubernetes Resources

Apply all Kubernetes configuration files:

```bash
kubectl apply -f deployment.yaml
kubectl apply -f service.yaml
kubectl apply -f configmap.yaml
```

---

### 3. Verify Cluster Resources

Check the status of nodes, pods, deployments, services, and ConfigMaps:

```bash
kubectl get nodes
kubectl get pods
kubectl get deployments
kubectl get services
kubectl get configmaps
```

---

### 4. Access the Application

Retrieve the service URL generated by Minikube:

```bash
minikube service flask-service --url
```

Open the generated URL in a web browser to verify successful application deployment.

---

### 5. Scale the Deployment

Increase the number of application replicas:

```bash
kubectl scale deployment flask-deployment --replicas=4
```

Reduce replicas back to two:

```bash
kubectl scale deployment flask-deployment --replicas=2
```

---

### 6. Inspect Resources and Logs

View deployment details:

```bash
kubectl describe deployment flask-deployment
```

Inspect Pod information:

```bash
kubectl describe pods
```

View container logs:

```bash
kubectl logs <pod-name>
```

---

### 7. Perform a Rolling Update

Update the application image without downtime:

```bash
kubectl set image deployment/flask-deployment \
flask-container=gcr.io/google-samples/hello-app:2.0
```

Monitor rollout progress:

```bash
kubectl rollout status deployment/flask-deployment
```

View rollout history:

```bash
kubectl rollout history deployment/flask-deployment
```

Rollback if required:

```bash
kubectl rollout undo deployment/flask-deployment
```

## Kubernetes Interview Questions & Answers

### 1. What is Kubernetes?

Kubernetes is an open-source container orchestration platform that automates the deployment, scaling, networking, and management of containerized applications.

### 2. What is the role of Kubelet?

Kubelet is an agent that runs on each Kubernetes node and ensures that containers defined in Pod specifications are running and healthy.

### 3. Explain Pods, Deployments, and Services.

**Pod**

* Smallest deployable unit in Kubernetes
* Encapsulates one or more containers

**Deployment**

* Manages Pod replicas
* Supports scaling and rolling updates
* Ensures desired state is maintained

**Service**

* Provides a stable network endpoint
* Enables communication with Pods
* Supports load balancing

### 4. How do you scale applications in Kubernetes?

```bash
kubectl scale deployment <deployment-name> --replicas=<count>
```

### 5. What is a Namespace?

A Namespace is a logical partition within a Kubernetes cluster used to organize and isolate resources between teams, projects, or environments.

### 6. What is the difference between ClusterIP, NodePort, and LoadBalancer?

| Service Type | Description                                                           |
| ------------ | --------------------------------------------------------------------- |
| ClusterIP    | Accessible only within the cluster                                    |
| NodePort     | Exposes application through a static port on each node                |
| LoadBalancer | Exposes application through a cloud provider's external load balancer |

### 7. What are ConfigMaps?

ConfigMaps are Kubernetes objects used to store non-sensitive configuration data as key-value pairs, allowing configuration to be separated from application code.

### 8. How are Rolling Updates performed?

Update the container image:

```bash
kubectl set image deployment/<deployment-name> \
<container-name>=<new-image>
```

Monitor deployment status:

```bash
kubectl rollout status deployment/<deployment-name>
```

Rollback to the previous version:

```bash
kubectl rollout undo deployment/<deployment-name>
```
